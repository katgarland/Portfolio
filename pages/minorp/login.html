Log In Screen

Project Details

  • Student Designer
  • 4041 Interface and Interaction: Design Experiences
  • Length: 2 weeks

Objectives

Our goal was to explore statefulness by designing at least 8 screens for a company's log-in. We were able to choose which company we designed for, so I chose to design a screen related to my Design Thesis project, which will be up on my portfolio at a later date. The project brief instructed to write if/then statements for our log in screen; you can view mine at the bottom of the page.

Conditional Statements

Text Input Instances

  • If a user hovers over a text box, the cursor will change from the default cursor to a “text” cursor.
  • If a user clicks on a text box, the outline of the textbox turns orange and continues to be highlighted until the user finishes the input.
  • If the user begins typing in the password box, the fill of the password box will go from transparent to black.
  • If the user stops typing in the password box and leaves content in the password box, the fill of the box will turn black at 50% opacity.

Password Instances

  • If the user begins typing in the password box, the eye icon will appear.
  • If the user hovers over the eye icon, the cursor will change from the default to a “pointer” cursor.
  • If the user clicks the closed eye icon, then the eye opens and the text in the Password field is revealed.
  • If the user clicks the open eye icon, then the eye closes and the text in the Password field is hidden.

Log In Instances

  • If a user hovers over the “Log In” button while the Username and Password fields are filled, the cursor will change from the default to a “pointer” cursor.
  • If a user hovers over the “Log In” button while a Username or Password field is blank, the blank field(s) will be outlined in red.
  • If a user fills both the Username and Password fields, then the “Log In” button goes from grey to yellow.
  • If the user hovers on the “Log In” button while it is yellow, it will turn orange.
  • If a user clicks “Log in” and both Username and Password correlate to a user's account, then the user is advanced to the next screen.
  • If a user clicks “Log in” and the Username does correlate to a user's account but the Password does not, the messages “Forgot Password?” and “Looking for a consumer log in?” are displayed at the bottom, and the “Log In” button will turn grey until the user makes a change to either text input field, and the Password field's background color will turn orange at a 20% opacity.
  • If the user clicks “Log In” and the Username does not correlate to a user's account, the error messages “Username is wrong. Contact admin?” and “Looking for a consumer log in?” will be displayed, and the login field's background color will turn orange at a 20% opacity.

Biometrics Instances

  • If the user hovers over the Biometrics icon, the fill will turn from orange to transparent.
  • If the user clicks the Biometrics icon and Biometrics are read successfully, the email and password are autofilled and the input box's background is set to yellow at 15% opacity, and the Icon will turn from the default face to the user's profile picture, and the Biometrics label will change to a Welcome Back message.
  • If the user clicks the Biometrics icon and there is a reading error, the Biometrics error message is displayed and the Biometrics icon will frown.

Stock Instances

  • If the user hovers over the stock, a tooltip will say “Today's Stock”

Forgot Password Instances

  • If a user clicks “Forgot password,” they are taken to a separate page to reset their password.

Remind me to break Instances

  • If the user clicks the “Remind to break” checkbox, the checkbox will turn orange and a check will appear.
  • If the user clicks the “Remind to break” checkbox, after logged in, the user will receive a pop-up notification to remind them to take a break after 4 hours.
